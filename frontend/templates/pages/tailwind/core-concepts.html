Hover, Focus e Outros States

Em Tailwind cada utilitário pode ser aplicado condicionalmente adicionando uma variante no começo da classe (por exemplo hover:bg-sky-700, focus:ring-2, sm:active:translate-y-0.5). Variantes podem ser empilhadas (ex.: dark:md:hover:bg-fuchsia-600). :contentReference[oaicite:1]{index=1}

:hover / :focus / :active

Casos básicos e combinações comuns para interações.

hover:bg-sky-700
focus:ring-2
active:translate-y-0.5
hover/focus/active combos
<div class="hover:bg-sky-700">...</div>
<div class="focus:ring-2 focus:ring-violet-300">...</div>
<div class="active:translate-y-0.5">...</div>

Estilos que respondem ao foco de filhos ou ao foco visível (teclado).

focus-within no container
focus-visible mostra foco apenas quando navegando por teclado

Respeite preferências do usuário para reduzir animações — combine com utilitários de transição e transform.

Dark Mode

Tailwind inclui a variante dark: para estilizar quando o tema escuro estiver ativo. Por padrão isso usa prefers-color-scheme, mas você pode trocar para ativação via classe ou atributo (útil para toggles manuais). Veja também exemplos de transições, componentes e padrões comuns. :contentReference[oaicite:1]{index=1}

Básico: dark: (prefers-color-scheme)

Sem configuração extra, Tailwind aplica dark: quando o usuário tem preferência por tema escuro.

Título — muda de cor em dark

Parágrafo com utilitários dark:.

<div class="bg-white dark:bg-gray-800">
    <h3 class="text-gray-900 dark:text-white">...</h3>
    <p class="text-gray-600 dark:text-gray-300">...</p>
</div>

Toggle manual via classe (html.dark)

Substitua a variante dark para aplicar quando a classe dark existe no <html> (ou em qualquer ancestor) — útil para toggles JS. :contentReference[oaicite:2]{index=2}

Com html.classList.toggle('dark') (server/client)

/* app.css */
@import "tailwindcss";
@custom-variant dark (&:where(.dark, .dark *));

/* HTML */
<html class="dark">
<body>
    <div class="bg-white dark:bg-black"> ... </div>
</body>
</html>

Dica: normalmente você adiciona/remove a classe dark no document.documentElement via JS e persiste a preferência em localStorage. Exemplo de snippet abaixo. :contentReference[oaicite:3]{index=3} 

/* toggle.js (exemplo simples) */
if (localStorage.theme === "dark" || (!("theme" in localStorage) && window.matchMedia("(prefers-color-scheme: dark)").matches)) {
    document.documentElement.classList.add("dark");
} else {
    document.documentElement.classList.remove("dark");
}

/* ao trocar manualmente */
function setDark(){ localStorage.theme = "dark"; document.documentElement.classList.add("dark"); }
function setLight(){ localStorage.theme = "light"; document.documentElement.classList.remove("dark"); }
function setSystem(){ localStorage.removeItem("theme"); /* fallback para prefers-color-scheme */ }

Toggle via data-attribute (ex.: data-theme="dark")

Você também pode ativar dark mode usando um atributo — basta ajustar o @custom-variant. :contentReference[oaicite:4]{index=4}

/* app.css */
@import "tailwindcss";
@custom-variant dark (&:where([data-theme=dark], [data-theme=dark] *));

/* HTML */
<html data-theme="dark">
<body>
    <div class="bg-white dark:bg-black"> ... </div>
</body>
</html>

Útil quando você quer controlar tema via atributos em vez de classes (server-rendering compatível).

Three-way toggle (light / dark / system)

Combine selector-based dark variant + window.matchMedia para suportar: forçar light, forçar dark, ou usar o sistema. :contentReference[oaicite:5]{index=5}

Exemplo visual: botão que alterna entre os três estados (persistência em localStorage recomendada).

// lógica resumida (ver docs para snippet completo)
document.documentElement.classList.toggle("dark", localStorage.theme === "dark" ||
(!("theme" in localStorage) && window.matchMedia("(prefers-color-scheme: dark)").matches));

Transições de tema & FOUC

Para evitar flash of unstyled content (FOUC) ao aplicar a classe dark via JS, injete um pequeno script inline no <head> que aplica o tema antes do paint. :contentReference[oaicite:6]{index=6}

/* Adicione inline no <head> para aplicar tema antes do paint (ex: minimal snippet) */
(function () {
    try {
        var theme = localStorage.getItem('theme');
        if (theme === 'dark' || (!theme && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
            document.documentElement.classList.add('dark');
        } else {
            document.documentElement.classList.remove('dark');
        }
    } catch (e) {}
})();

Variáveis CSS e tema customizado

Combine CSS vars com classes Tailwind ou estenda a palette via tailwind.config.js.

Exemplo com variável --- usar rgb(var(--brand) / 1) em CSS ou adicionar ao tema.

/* tailwind.config.js (exemplo) */
module.exports = {
    theme: {
        extend: {
            colors: {
                brand: {
                    50: '#fff5f7',
                    500: '#ff6384',
                    700: '#e6516b',
                }
            }
        }
    }
}

/* CSS inline usando var */
.my-el { background-color: rgb(var(--brand) / 1); }
